Part 1 — The Danger of Motors

Driver Installation Guide

The “Muscle” vs The “Brain”

We learned that the Arduino is the Brain (it sends 0s and 1s using low-voltage signals).

Motors are the Muscles. They require a large flow of electricity (current) to generate magnetic fields strong enough to spin wheels.

  • LED Requires: ~20 milliAmps (mA)
  • Arduino Pin Maximum: ~40 milliAmps (mA)
  • DC Motor Requires: ~500 to 1,000+ milliAmps (mA)

Rule #1 of Robotics

Never Power a Motor Directly From an Arduino Pin!

If you try to draw 1,000 mA from a pin designed to supply only 40 mA, you will instantly melt the internal circuitry of the Arduino chip.

Think of it like trying to fill a swimming pool using a drinking straw. The pressure will destroy the straw.

Motor Logic: Polarity & Direction

The Manual Problem

Can we swap wires while driving?

To make a robot reverse, we would have to stop it, unplug the wires, swap them, and plug them back in.

This is impossible for a computer to do by itself… unless we use electronic switches.

The Solution: The H-Bridge

An H-Bridge is a circuit of 4 electronic switches. By opening and closing different pairs, we can “redirect” the electricity flow without touching a single wire!

  • To go Forward: Close S1 and S4.
  • To go Backward: Close S2 and S3.
  • To Stop: Open all switches.

Caution: Never close S1 and S3 at the same time (Short Circuit!)

The Real H-Bridge

Building an H-Bridge from tiny transistors is hard.

Instead, we use the L298N Module. It’s a “dual” driver that can control two motors.

It gives us 3 types of pins:

  • IN Pins: Controls Direction (Forward/Backward)
  • EN Pins: Controls Speed (The Gas Pedal!)

Tinkercad Note

Tinkercad does not have the L298N “Red Board.” Instead, we will use the L293D chip, which is the H-Bridge “brain” inside these modules. The logic is exactly the same!

Part 2 — The Physical Build

Transition to Reality

Now it’s time to build the real thing.

Safety First

Do not plug in the battery pack until the instructor inspects your wiring. A short circuit on the battery pack can cause it to overheat.

Step 1: Chassis Assembly

The chassis is the physical body of the robot.

  1. Mount the DC motors to the frame using the provided brackets.
  2. Attach the wheels to the motor shafts.
  3. Secure the caster wheel (the tiny front wheel) to the front of the frame.
  4. Mount the Arduino and the L298N Motor Driver to the top deck.

Step 2: Power Connections

We must route the power carefully.

From the Battery Pack:

  • Red wire (+) \(\rightarrow\) Motor Driver 12V
  • Black wire (-) \(\rightarrow\) Motor Driver GND

From the Arduino:

  • Arduino GND \(\rightarrow\) Motor Driver GND
  • Do not connect Arduino 5V to the motor power!

A Simpler Way to Drive the Motor

Standard L298N

  • 2 Direction pins (IN1, IN2)
  • 1 Enable pin (Speed)
  • \(\rightarrow\) 3 logic wires / motor

Our Kit’s Driver

  • Built-in logic inverter
  • Signals are always opposite
  • \(\rightarrow\) 2 logic wires / motor

Official Kit Wiring Guide

The 2-Wire Logic

How it works

  1. Direction (Ctrl):
    • HIGH = Forward
    • LOW = Backward
  2. Speed (PWM):
    • 0 (Stop) to 255 (Full Speed)

Note

Trade-off: Saves Arduino pins for sensors, but removes the H-bridge “brake” mode (it coasts to a stop).

Code Example

// Move Left Motor Forward
digitalWrite(ML_Ctrl, HIGH);
analogWrite(ML_PWM, 200); 

// Move Left Motor Backward
digitalWrite(ML_Ctrl, LOW);
analogWrite(ML_PWM, 200);

Step 3 & 4: Final Wiring

1. Motor Outputs

Connect motors to the driver:

  • Left: OUT1 & OUT2
  • Right: OUT3 & OUT4

Tip: Check the wiring of the motor; if it spins backward, swap its two wires (especially if using a custom kit).

2. Logic Inputs

Connect Arduino to Driver:

Motor Direction (Ctrl) Speed (PWM)
Left Pin 4 Pin 5
Right Pin 2 Pin 6

Why these pins?

The PWM pins (5 and 6) are marked with ~ on your Arduino. These are the only pins that can control speed!

Part 3 — Programming Movement

The Base Code

To drive a car, we must manage 4 pins simultaneously.

// Define our pins
#define ML_Ctrl 4    // Left Direction
#define ML_PWM 5     // Left Speed
#define MR_Ctrl 2    // Right Direction
#define MR_PWM 6     // Right Speed

void setup() {
  pinMode(ML_Ctrl, OUTPUT);
  pinMode(ML_PWM, OUTPUT);
  pinMode(MR_Ctrl, OUTPUT);
  pinMode(MR_PWM, OUTPUT);
}

Functions: Organizing Logic

Typing digitalWrite four times just to move forward is tedious. In C++, we can create custom functions to bundle commands together.

void car_front() {
  digitalWrite(ML_Ctrl, HIGH);
  analogWrite(ML_PWM, 150); // Set speed (0-255)
  digitalWrite(MR_Ctrl, HIGH);
  analogWrite(MR_PWM, 150);
}

void car_Stop() {
  analogWrite(ML_PWM, 0);
  analogWrite(MR_PWM, 0);
}

Driving the Car

Now, our loop() becomes very simple and easy to read!

void loop() {
  car_front();
  delay(2000); // Drive for 2 seconds
  
  car_Stop();
  delay(1000); // Wait 1 second
}

How does a tank turn? It spins its tracks in opposite directions!

void car_right() {
  digitalWrite(ML_Ctrl, HIGH); // Left Forward
  analogWrite(ML_PWM, 150);

}

Part 4 — How Does a TV Remote Work?

Invisible Control

How does a TV remote talk to the TV? It blinks a light that we can’t see!

  • Infrared (IR) is light with a frequency lower than red light.
  • Our eyes can’t see it, but the robot’s secondary “electronic eye” can.
  • It’s like a flashlight sending a secret Morse code at high speed.

The Hardware

To receive signals, we use an IR Receiver Module.

Connections:

  1. GND → GND
  2. VCC → 5V
  3. Signal → Pin 3 (Digital)

Activity: IR Intuition

Can you see the invisible light?

  1. Take your phone camera.
  2. Point the IR Remote at the camera lens.
  3. Press a button.
  4. Look at the screen! Do you see a purple/pink blink?

Why? Digital cameras can “see” slightly more of the spectrum than human eyes.

Add Library as ZIP

Protocol Check

The Keyestudio remote uses the NEC Protocol. The library will handle this for us automatically!

Activity: Scanning Code

Before we control the car, we need to know what “numbers” your buttons send.

  1. Open a new sketch in Arduino IDE and paste this code:
#include <IRremote.hpp>

int RECV_PIN = 3;

void setup() {
    Serial.begin(9600);
    IrReceiver.begin(RECV_PIN, ENABLE_LED_FEEDBACK); 
}

void loop() {
    if (IrReceiver.decode()) {
        if (!(IrReceiver.decodedIRData.flags & IRDATA_FLAGS_IS_REPEAT)) {
            Serial.print("Command: 0x");
            Serial.println(IrReceiver.decodedIRData.command, HEX);
        }
        IrReceiver.resume(); 
    }
}

Activity: Mapping the Remote

  1. Upload the code and open the Serial Monitor (9600 baud).
  2. Press buttons and write down the HEX codes for:
    • Up Arrow (▲)
    • Down Arrow (▼)
    • Left Arrow (◄)
    • Right Arrow (►)
    • OK / Stop

What are those codes?

When you press a button, the remote sends a command number like 0x46.

  • This is a HEXADECIMAL number (Base-16).
  • It’s like a unique ID for that specific button.
  • If you use a different remote, the numbers will be different!

Part 6 — Mapping Buttons to Directions

The Logic: if...else Statements

To handle multiple buttons, we can use an if...else if structure since it’s easier to read.

void handleCommand(uint16_t cmd) {
  if (cmd == 0x46) { // Replace with YOUR 'UP' code
    car_front();
  } else if (cmd == 0x15) { // Replace with YOUR 'DOWN' code
    car_back();
  } else if (cmd == 0x40) { // Replace with YOUR 'OK' code
    car_Stop();
  }
}

Setup & Loop

We need to tell the robot to “listen” at the start and “resume” after every signal.

#include <IRremote.hpp>
int RECV_PIN = 3;

void setup() {
  Serial.begin(9600);
  IrReceiver.begin(RECV_PIN, ENABLE_LED_FEEDBACK);
  // ... usual motor setup goes here ...
}

void loop() {
  if (IrReceiver.decode()) {
    uint16_t cmd = IrReceiver.decodedIRData.command;
    Serial.println(cmd, HEX); 
    
    handleCommand(cmd); // Run our logic
    
    IrReceiver.resume(); // Ready for next signal
  }
}

Why Do We Need Auto-Stop?

The Problem 🏎️💨

If you press a button and let go, the car keeps moving forever!

  • The remote only sends signals while pressed.
  • It never sends a “stop” signal when released.

The Solution ⏱️

Use Arduino’s built-in stopwatch: millis().

  1. Save Time: Record the time of every IR signal.
  2. Check Time: If >200ms passes with no new signal, tell the car to stop.

2. Advanced Code (With Auto-Stop)

Now let’s see how that looks in our code. Pay attention to lastSignalTime and the millis() function.

#include <IRremote.hpp> // Use .hpp for version 4.x+

#define ML_Ctrl 4    
#define ML_PWM 5   
#define MR_Ctrl 2    
#define MR_PWM 6   

int RECV_PIN = 3;
int carSpeed = 100; 

// --- NEW VARIABLES FOR TIMEOUT ---
unsigned long lastSignalTime = 0;   // Stores the time of the last IR signal
const int timeoutThreshold = 200;   // Milliseconds to wait before stopping (IR repeats ~110ms)

void handleCommand(uint16_t command) {
    if (command == 0x46) { // Replace with YOUR 'UP' code
        car_front();
    } else if (command == 0x15) { // Replace with YOUR 'DOWN' code
        car_back();
    } else if (command == 0x44) { // Replace with YOUR 'LEFT' code
        car_left();
    } else if (command == 0x43) { // Replace with YOUR 'RIGHT' code
        car_right();
    } else if (command == 0x40) { // Replace with YOUR 'OK' code
        car_Stop();
    } else if (command == 0x16) { // Speed Up
        carSpeed = min(carSpeed + 20, 255);
    } else if (command == 0x19) { // Speed Down
        carSpeed = max(carSpeed - 20, 0);
    }
}

void setup() {
    pinMode(ML_Ctrl, OUTPUT);
    pinMode(ML_PWM, OUTPUT);
    pinMode(MR_Ctrl, OUTPUT);
    pinMode(MR_PWM, OUTPUT);
    
    Serial.begin(9600);
    IrReceiver.begin(RECV_PIN, ENABLE_LED_FEEDBACK); 
}

void loop() {
    if (IrReceiver.decode()) {
        // If it's a repeat signal, just update the timer to keep the car moving
        if (IrReceiver.decodedIRData.flags & IRDATA_FLAGS_IS_REPEAT) {
            lastSignalTime = millis();
        } 
        else {
            // It's a new button press
            uint16_t command = IrReceiver.decodedIRData.command;
            lastSignalTime = millis(); // Update timer
            
            handleCommand(command);
        }
        IrReceiver.resume(); 
    }

    // --- AUTOMATIC STOP LOGIC ---
    // If current time is 200ms past the last signal, stop the car
    if (millis() - lastSignalTime > timeoutThreshold) {
        car_Stop();
    }
}

// --- Movement Functions ---
void car_front() {
    digitalWrite(ML_Ctrl, HIGH);
    analogWrite(ML_PWM, carSpeed);
    digitalWrite(MR_Ctrl, HIGH);
    analogWrite(MR_PWM, carSpeed);
}

void car_back() {
    digitalWrite(ML_Ctrl, LOW);
    analogWrite(ML_PWM, carSpeed);
    digitalWrite(MR_Ctrl, LOW);
    analogWrite(MR_PWM, carSpeed);
}

void car_left() {
    digitalWrite(ML_Ctrl, LOW);
    analogWrite(ML_PWM, carSpeed);
    digitalWrite(MR_Ctrl, HIGH);
    analogWrite(MR_PWM, carSpeed);
}

void car_right() {
    digitalWrite(ML_Ctrl, HIGH);
    analogWrite(ML_PWM, carSpeed);
    digitalWrite(MR_Ctrl, LOW);
    analogWrite(MR_PWM, carSpeed);
}

void car_Stop() {
    analogWrite(ML_PWM, 0);
    analogWrite(MR_PWM, 0);
}

Part 9 — The Bluetooth Module

Cutting the Cord

Instead of an infrared remote, we are going to use Bluetooth! This means we can control our robot using a smartphone app, just like a real RC car or a drone.

We will use an HM-10 Bluetooth module and the Dabble App, which transforms your smartphone into a virtual controller.

Wiring the Module

Here is the trick with communication pins: TX must talk to RX.

  1. Connect VCC to Arduino 5V.
  2. Connect GND to Arduino GND.
  3. Connect TXD to Arduino RX (Pin 0).
  4. Connect RXD to Arduino TX (Pin 1).

Important

Because we use Pins 0 and 1, you must disconnect the TX and RX wires when uploading code!

Installing the Dabble Library

Before we can write our code, we need to add the Dabble library to the Arduino IDE:

📥 Download: Dabble-Arduino.zip

In the Arduino IDE, go to Sketch → Include Library → Add .ZIP Library… and select the file you just downloaded.

Adding the Library

Install Dabble in your Smartphone

Download the Dabble App from your App Store. Once installed, open the app, click the connect icon, and pair it with your HM-10 module (Password is usually 1234 or 000000).

Part 10 — App Driving

The Dabble Gamepad

We will use the Gamepad Module in the Dabble app to drive our robot. The Gamepad has buttons like Up, Down, Left, and Right.

Dabble Gamepad Module

Complete Bluetooth Code

What’s New?

  1. Include Dabble: #include <Dabble.h> and Dabble.begin(Serial)
  2. Read Gamepad: Dabble.processInput() and GamePad.isUpPressed()

The movement functions (car_front(), etc.) remain exactly the same as the IR remote!

#define CUSTOM_SETTINGS
#define INCLUDE_GAMEPAD_MODULE
#include <Dabble.h>

#define ML_Ctrl 4    
#define ML_PWM 5   
#define MR_Ctrl 2    
#define MR_PWM 6   

int carSpeed = 100; 

void setup() {
    pinMode(ML_Ctrl, OUTPUT);
    pinMode(ML_PWM, OUTPUT);
    pinMode(MR_Ctrl, OUTPUT);
    pinMode(MR_PWM, OUTPUT);
    
    Serial.begin(9600);
    Dabble.begin(Serial); // Start Bluetooth using Hardware Serial
}

void handleCommand() {
    if (GamePad.isUpPressed()) {
        car_front();
    } 
    else if (GamePad.isDownPressed()) {
        car_back();
    }
    else if (GamePad.isLeftPressed()) {
        car_left();
    }
    else if (GamePad.isRightPressed()) {
        car_right();
    }
    else {
        car_Stop();
    }
}

void loop() {
    Dabble.processInput(); // Refresh data from the smartphone
    handleCommand();
}

Part 12 — Problem Patterns

What can a moving robot actually do?

A remote-controlled car is incredibly fun — but useful robots solve real problems on their own.

  • 💊 Deliver medicine inside a hospital
  • 🔍 Inspect dangerous pipes underground
  • 📦 Move boxes in a warehouse
  • 🚜 Navigate a farm autonomously

To do this safely, they need more than a human holding a remote control!

The Pattern Behind Every Robot

Every robot program follows the same 3-step loop:

1️⃣ Sense

Read the world — buttons, sensors, signals.

2️⃣ Decide

Run logic — if...else, functions.

3️⃣ Act

Drive motors, blink LEDs, make sounds.

Today you completed Act (Motors) and basic Sense (IR Receiver).
